# Проект 1. Калькулятор в терминале

**Длительность:** 2 недели
**Цель:** научиться писать законченные программы на Go, освоить базовый синтаксис, прочувствовать идиоматичную обработку ошибок.

## Что в итоге будет уметь программа

Принимать два числа и операцию, выводить результат. Не падать ни на каком вводе. Запускаться из терминала командой вида `./calc 5 + 3`.

## Структура проекта

| Модуль | Название | Главная тема | Длительность |
|--------|----------|--------------|--------------|
| 1 | Захардкоженный калькулятор | Переменные, типы, арифметика, вывод | 2 дня |
| 2 | Ввод от пользователя | `fmt.Scan`, указатели, stdin | 2 дня |
| 3 | Аргументы командной строки | `os.Args`, `strconv`, конвертация типов | 2 дня |
| 4 | Обработка ошибок | `error` как значение, идиома `result, err` | 3 дня |
| 5 | Расширение операций | `switch`, декомпозиция на функции | 3 дня |

## Как работать с модулями

Каждый модуль = папка с четырьмя файлами:

- **`theory.md`** — теория, которую нужно понять перед практикой. Читай вдумчиво, не пролистывай.
- **`condition.md`** — конкретное задание. Что именно нужно написать.
- **`hints.md`** — наводящие вопросы, если застрял. **Открывай только после того, как сам поборолся минимум 30 минут.**
- **`checklist.md`** — маркеры готовности модуля. Сверься перед переходом к следующему.

## Правило перехода между модулями

Не переходи к следующему модулю, пока не закрыл **все** пункты checklist текущего. Это не формальность — каждый следующий модуль опирается на навыки предыдущего.
