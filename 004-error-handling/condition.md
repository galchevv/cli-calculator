# Модуль 4. Обработка ошибок — условие

## Задание

Доведи свой калькулятор до состояния, в котором он **не падает ни на каком вводе** и при любой ошибке выводит **понятное сообщение**, что именно пошло не так.

## Что должна делать программа

Все нормальные случаи работают как раньше:

```bash
$ go run main.go 20 + 4
Результат: 24
```

Все ошибочные случаи **обрабатываются явно**:

```bash
$ go run main.go
Использование: calc <число> <операция> <число>
$ echo $?
1

$ go run main.go 20 +
Использование: calc <число> <операция> <число>

$ go run main.go abc + 5
Первый аргумент не число: abc

$ go run main.go 10 + xyz
Третий аргумент не число: xyz

$ go run main.go 10 / 0
Ошибка: деление на ноль

$ go run main.go 10 ^ 5
Неизвестная операция: ^
```

## Требования

1. Проверка количества аргументов — на самой первой строке `main` после объявлений.
2. Проверка `err` после **каждого** `strconv.ParseFloat`.
3. Проверка `b == 0` **до** деления (а не после, когда уже получил `+Inf`).
4. Проверка неизвестной операции через `default` в `switch`.
5. Все сообщения об ошибках идут в `os.Stderr` (через `fmt.Fprintln` или `fmt.Fprintf`).
6. После каждого сообщения об ошибке — `os.Exit(1)`.
7. Никакого `_` для игнорирования ошибок. Все ошибки проверяются явно.

## Шаги

1. Возьми код из модуля 3
2. Начни сверху: проверка `len(os.Args)`. Если не подходит — Stderr + Exit(1)
3. Замени `n, _ := strconv.ParseFloat(...)` на `n, err := strconv.ParseFloat(...)` + проверка
4. В ветке деления добавь проверку на ноль
5. В `switch` добавь `default`
6. Прогоняй каждый случай из примеров выше — проверь, что всё работает

## Эксперименты

1. **Запусти `echo $?`** после каждого случая (на Linux/macOS). При успехе должен быть `0`, при ошибке — `1`. Это и есть «код выхода», его читают скрипты.
2. **Перенаправь stderr в файл:** `go run main.go abc + 5 2> errors.log`. Открой `errors.log` — там должно быть сообщение об ошибке. А на экране — пусто. Это и есть смысл разделения stdout и stderr.
3. **Перенаправь stdout в файл:** `go run main.go 20 + 4 > result.txt`. На экране пусто, в файле — `Результат: 24`.
4. **Попробуй оба сразу:** `go run main.go abc + 5 > result.txt 2> errors.log`. Какой файл окажется пустым?
