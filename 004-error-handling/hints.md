## Уровень 2: про `if err != nil`

Это **главный паттерн Go**, который ты будешь писать тысячи раз:

```go
result, err := someFunc()
if err != nil {
    // обработка
    return  // или os.Exit(1) в main
}
// здесь result уже можно безопасно использовать
```

Заметь два момента:

- Сравниваем с `nil`, не с пустой строкой и не с `0`. `error` — это интерфейс, его «отсутствие» — это `nil`.
- После проверки идёт «обработка ошибки». Обычно это либо вернуть ошибку выше (`return ...`), либо завершить программу (`os.Exit(1)`).

## Уровень 3: про переиспользование `err`

Если у тебя в одной функции несколько вызовов с `err`, можно переиспользовать одно имя:

```go
a, err := strconv.ParseFloat(os.Args[1], 64)
if err != nil { ... }

b, err := strconv.ParseFloat(os.Args[3], 64)  // err уже существует, переиспользуем
if err != nil { ... }
```

Технически Go разрешает использовать `:=` повторно, если **хотя бы одна** новая переменная появляется слева. Здесь `b` — новая, поэтому работает.

## Уровень 4: про stderr

Если ты пишешь так:

```go
fmt.Println("ошибка: что-то не так")
```

— это идёт в `stdout`. Для ошибок надо в `stderr`:

```go
fmt.Fprintln(os.Stderr, "ошибка: что-то не так")
```

`Fprintln` (буква F — от **F**ile) — это `Println`, который пишет не на экран по умолчанию, а в указанный поток. `os.Stderr` — это и есть поток ошибок.

`Fprintf` — то же самое, но с форматированием:

```go
fmt.Fprintf(os.Stderr, "не могу прочитать число: %s\n", os.Args[1])
```

Не забывай `\n` в конце у `Fprintf` — он, в отличие от `Fprintln`, не добавляет перевод строки сам.

## Уровень 5: про деление на ноль для float64

Если ты ожидал, что `10 / 0` для `float64` упадёт — нет, Go этого не делает. Он возвращает `+Inf` (положительная бесконечность):

```
Результат: +Inf
```

Это математически корректно (предел `1/x` при `x → 0+` равен `+Inf`), но для пользователя калькулятора — мусор. Ловить нужно **до** деления:

```go
case "/":
    if b == 0 {
        fmt.Fprintln(os.Stderr, "Ошибка: деление на ноль")
        os.Exit(1)
    }
    result = a / b
```

## Уровень 6: про default в switch

Если в `switch` нет ветки `default`, неизвестные значения **просто игнорируются** — никакой ошибки не будет, программа пойдёт дальше с нулевым результатом. Это и есть тихий баг.

```go
switch op {
case "+":
    result = a + b
// ...
default:
    fmt.Fprintf(os.Stderr, "Неизвестная операция: %s\n", op)
    os.Exit(1)
}
```

`default` срабатывает, если ни одна другая ветка не подошла. Это твой «всё остальное».
