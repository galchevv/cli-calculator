# Модуль 3. Аргументы командной строки — теория

## Контекст

В модуле 2 программа спрашивает данные через `Scan` — это интерактивный режим. Это удобно для человека, но **неудобно для автоматизации**.

Представь, что ты хочешь запустить калькулятор внутри скрипта или другой программы. Тебе придётся как-то «вводить» данные за пользователя — это сложно и хрупко.

Гораздо удобнее запускать калькулятор так:

```bash
./calc 20 + 4
```

Программа сразу видит все нужные данные, считает и выводит результат. Никакого диалога, никакого ожидания.

Это и есть **аргументы командной строки** — данные, которые программа получает в момент запуска.

## Ядро: os.Args

В Go все аргументы, переданные программе при запуске, лежат в переменной `os.Args`. Это **слайс строк** — упорядоченный набор значений типа `string`.

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    fmt.Println(os.Args)
}
```

Запусти это так: `go run main.go hello world 42`

Выведет что-то вроде:

```
[/tmp/go-build123/exe/main hello world 42]
```

Что мы видим:

- `os.Args[0]` — это **путь к самой программе**. Не аргумент в привычном смысле, а служебная информация.
- `os.Args[1]` — первый «настоящий» аргумент (`hello`)
- `os.Args[2]` — второй (`world`)
- `os.Args[3]` — третий (`42`)

## Почему `os.Args[0]` — это имя программы

Это историческая традиция Unix. Программа должна знать своё собственное имя — например, чтобы вывести его в сообщении об ошибке: `calc: error: division by zero`. Поэтому ОС всегда подставляет имя нулевым аргументом.

Тебе на этом этапе оно не нужно, но **никогда не забывай, что осмысленные аргументы начинаются с индекса 1**, а не 0. Это одна из самых частых ловушек новичка.

## Структура: как обращаться к элементам слайса

Слайсы в Go похожи на массивы из других языков. К элементу обращаются по индексу через квадратные скобки:

```go
firstArg := os.Args[1]   // строка
secondArg := os.Args[2]  // строка
```

**Важно:** все элементы `os.Args` — **строки**. Даже если ты передал число `42`, в слайсе оно лежит как строка `"42"`. Чтобы делать с ним арифметику, нужно сначала **превратить строку в число**.

## Конвертация строк в числа: пакет strconv

Пакет `strconv` (от **str**ing **conv**ersion) умеет превращать строки в числа и обратно. Тебе нужны две функции:

```go
import "strconv"

// Строка → целое число
n, err := strconv.Atoi("42")  // n = 42, err = nil

// Строка → дробное число
f, err := strconv.ParseFloat("3.14", 64)  // f = 3.14, err = nil
```

Заметь: обе функции возвращают **два значения**. Первое — результат, второе — ошибка. Это идиома Go, к которой ты привыкнешь до автоматизма: «делаю что-то, что может не получиться → получаю результат и ошибку».

Если строку **нельзя** превратить в число (например, `"abc"`), результат будет `0`, а в `err` появится осмысленное сообщение.

Полностью обработкой ошибок мы займёмся в модуле 4. Пока в этом модуле **достаточно знать**, что вторую переменную нужно объявить, и компилятор не разрешит её игнорировать. Если хочешь временно «забить» — используй знак подчёркивания:

```go
n, _ := strconv.Atoi("42")  // ошибку явно игнорируем
```

Знак `_` называется **blank identifier** — «слепая переменная». Go считает, что значение использовано, и не ругается.

## Длина слайса и проверка

Перед тем как обращаться к `os.Args[3]`, нужно убедиться, что в слайсе **есть** третий элемент. Иначе программа упадёт с паникой `index out of range`.

Длину слайса возвращает функция `len`:

```go
if len(os.Args) < 4 {
    fmt.Println("Использование: calc <число> <операция> <число>")
    os.Exit(1)
}
```

`os.Exit(1)` принудительно завершает программу с кодом ошибки. Код `0` — успех, любой другой — ошибка. Это тоже Unix-традиция.

## Пример с разбором

```go
package main

import (
    "fmt"
    "os"
    "strconv"
)

func main() {
    if len(os.Args) < 4 {
        fmt.Println("Использование: calc <число> <операция> <число>")
        os.Exit(1)
    }

    a, _ := strconv.ParseFloat(os.Args[1], 64)
    op := os.Args[2]
    b, _ := strconv.ParseFloat(os.Args[3], 64)

    var result float64
    switch op {
    case "+":
        result = a + b
    case "-":
        result = a - b
    case "*":
        result = a * b
    case "/":
        result = a / b
    }

    fmt.Println("Результат:", result)
}
```

Запуск:

```bash
go run main.go 20 + 4
```

Вывод:

```
Результат: 24
```

## Маленькая хитрость терминала

Если ты попробуешь запустить с `*` (умножение):

```bash
go run main.go 20 * 4
```

— скорее всего получишь странное поведение. Это потому, что `*` для терминала — **особый символ** (он подставляет туда список файлов из текущей папки). Чтобы передать звёздочку как обычную строку, оборачивай в кавычки:

```bash
go run main.go 20 "*" 4
```

Это не баг Go, это особенность Unix-shell. Полезный урок: **терминал — это тоже программа**, и у неё есть свои правила.
