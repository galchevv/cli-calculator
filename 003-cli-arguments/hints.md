# Модуль 3. Аргументы командной строки — подсказки

## Уровень 1: наводящие вопросы

1. В какой переменной живут аргументы командной строки?
2. Какого типа значения в этом слайсе?
3. Что нужно сделать со строкой, чтобы получить из неё число?
4. Какая функция возвращает длину слайса?

## Уровень 2: про индекс 0

Если ты начал брать первое число из `os.Args[0]` — программа ведёт себя странно или падает. Это потому, что в нулевом элементе лежит **путь к самой программе**, а не твой аргумент.

Запомни простое правило: **твои аргументы начинаются с индекса 1**. Это контринтуитивно (в нумерации с нуля как-то логичнее, чтобы первый аргумент был нулевым), но это стандарт Unix, который соблюдают все языки.

## Уровень 3: про strconv

Если ты пытаешься так:

```go
a := os.Args[1]
result := a + b
```

— получишь ошибку `mismatched types string and float64`. Потому что `os.Args[1]` — это **строка**, а не число. Складывать строку с числом нельзя.

Нужно сначала **превратить** строку в число. Для дробных чисел это делает `strconv.ParseFloat`:

```go
a, _ := strconv.ParseFloat(os.Args[1], 64)
```

Второй аргумент `64` — это «битность» результата. Для `float64` всегда передавай `64`. Для `float32` передал бы `32`, но он почти не используется.

## Уровень 4: про обязательные две переменные

Если ты пытаешься так:

```go
a := strconv.ParseFloat(os.Args[1], 64)
```

— получишь ошибку `multiple-value strconv.ParseFloat() in single-value context`. Это значит: «функция возвращает **два** значения, а ты пытаешься положить их в одну переменную».

Решение — принять оба значения. Если ошибка пока не нужна, используй `_`:

```go
a, _ := strconv.ParseFloat(os.Args[1], 64)
```

## Уровень 5: про проверку длины

Если ты обращаешься к `os.Args[1]` без проверки и запускаешь программу без аргументов — она упадёт с паникой:

```
panic: runtime error: index out of range [1] with length 1
```

Расшифровка: «ты попросил элемент с индексом 1, а в слайсе всего 1 элемент» (тот самый нулевой — путь к программе).

Проверка делается так:

```go
if len(os.Args) < 4 {
    fmt.Println("Использование: calc <число> <операция> <число>")
    os.Exit(1)
}
```

Почему `< 4`, а не `< 3`? Считаем: `os.Args[0]` (путь) + 3 аргумента = 4 элемента в слайсе. Если меньше — чего-то не хватает.

## Уровень 6: про звёздочку

Когда ты запускаешь `go run main.go 7 * 8` — терминал заменяет `*` на список всех файлов в текущей папке, и твоя программа получает не то, что ты ожидал. Это **shell-globbing** — особенность Unix-shell.

Решения два:
1. Оборачивать в кавычки: `go run main.go 7 "*" 8`
2. Экранировать обратным слэшем: `go run main.go 7 \* 8`

Оба варианта говорят shell: «не трогай этот символ, передай как есть».
