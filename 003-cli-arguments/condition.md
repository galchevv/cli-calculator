# Модуль 3. Аргументы командной строки — условие

## Задание

Перепиши калькулятор так, чтобы он принимал данные **из аргументов командной строки**, а не через интерактивный ввод.

## Что должна делать программа

```bash
$ go run main.go 20 + 4
Результат: 24

$ go run main.go 100 - 25
Результат: 75

$ go run main.go 7 "*" 8
Результат: 56

$ go run main.go 10 / 4
Результат: 2.5
```

## Требования

1. Программа берёт три аргумента: число, операцию, число.
2. Если аргументов меньше трёх — выводит подсказку по использованию и завершается.
3. Поддерживаются 4 операции: `+`, `-`, `*`, `/`.
4. Используй `strconv.ParseFloat` для конвертации.
5. Используй `switch` для выбора операции (не цепочку `if`).
6. Никакого `Scan` — программа не должна ничего спрашивать у пользователя.

## Шаги

1. Замени все `var x float64` + `Scan(&x)` на чтение из `os.Args`
2. Не забудь импортировать пакеты `os` и `strconv`
3. Помни: `os.Args[0]` — это имя программы, числа начинаются с `os.Args[1]`
4. Перед обращением к индексам — проверь длину слайса через `len`
5. Если что-то не так — выведи подсказку и `os.Exit(1)`
6. Если у тебя ещё цепочка `if`/`else if` — самое время переписать на `switch`

## Эксперименты

1. **Запусти без аргументов:** `go run main.go` — получи свою подсказку
2. **Запусти с двумя аргументами:** `go run main.go 10 +` — должна быть та же подсказка
3. **Запусти с буквами:** `go run main.go abc + def` — что произойдёт? Какой результат?
4. **Запусти с лишним аргументом:** `go run main.go 10 + 5 7` — что произойдёт?
5. **Запусти со звёздочкой без кавычек:** `go run main.go 7 * 8` — что произойдёт? Сравни с `go run main.go 7 "*" 8`.

## Вопрос для размышления

После эксперимента 3 ты увидишь, что программа выведет `Результат: 0`. Это **тихий баг** — программа не падает, но и не делает то, что ожидал пользователь. Подумай: **где** в коде это нужно было поймать? Чем это плохо? Запиши свои мысли — в модуле 4 мы будем это лечить.
