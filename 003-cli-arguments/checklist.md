# Модуль 3. Аргументы командной строки — чек-лист готовности

## Поведенческие маркеры

- [ ] Программа запускается командой `go run main.go <число> <операция> <число>`
- [ ] Никакого `Scan` в коде больше нет
- [ ] При запуске без аргументов выводится подсказка по использованию
- [ ] При неполных аргументах (только 1 или 2) — то же самое
- [ ] Все 4 операции работают: `+`, `-`, `*`, `/`
- [ ] Используется `switch`, а не цепочка `if/else if`
- [ ] Конвертация через `strconv.ParseFloat`
- [ ] Код закоммичен в git

## Самопроверка

Закрой код. Ответь:

1. Сколько раз в коде ты использовал `os.Args`?
2. Сколько раз ты использовал `strconv.ParseFloat`?
3. Где у тебя проверка длины слайса — в начале функции или в конце? Почему?

Если ты честно ответил «в начале, потому что иначе программа упадёт раньше, чем дойдёт до проверки» — отлично, ты понял идею **fail fast**. Это важный принцип, который нам ещё пригодится.
