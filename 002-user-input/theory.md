# Модуль 2. Ввод от пользователя — теория

## Контекст

В первом модуле ты научился выполнять операции с числами, **жёстко прописанными в коде**. Это полезно как тренировка, но как настоящий калькулятор работать не будет — каждый раз менять числа в исходнике и пересобирать программу абсурдно.

В этом модуле программа научится **разговаривать с пользователем**: спрашивать у него числа и операцию через терминал, а потом считать.

## Ядро: поток ввода (stdin) и поток вывода (stdout)

У каждой программы в Linux/macOS/Windows есть три стандартных потока:

- **stdin** (standard input) — поток, откуда программа **читает** данные. По умолчанию это клавиатура.
- **stdout** (standard output) — поток, куда программа **пишет** данные. По умолчанию это экран терминала.
- **stderr** (standard error) — отдельный поток для ошибок. Тоже идёт в терминал, но логически отделён.

Когда ты пишешь `fmt.Println("привет")`, Go берёт строку «привет» и кладёт её в **stdout**. Терминал её показывает на экране.

Когда ты хочешь **прочитать** что-то от пользователя — нужно достать данные из **stdin**. Для этого есть функция `fmt.Scan` и её родственники.

## Почему так

Это разделение — не прихоть Go, а универсальный паттерн всех Unix-систем. Он даёт огромную гибкость: ты можешь, например, перенаправить вывод одной программы во ввод другой через символ `|`:

```bash
echo "10 5" | ./calc
```

В этом случае программа `./calc` думает, что пользователь напечатал `10 5` с клавиатуры, хотя на самом деле эти данные пришли из вывода `echo`. Программа об этом не знает и знать не должна — она просто читает из stdin.

## Структура: как читать ввод

Базовая функция — `fmt.Scan`. Она читает значения из stdin и **записывает их в переменные**, которые ты ей передал.

```go
var x int
fmt.Scan(&x)
```

Здесь два важных момента:

1. **Переменная объявлена ДО** вызова `Scan`. `Scan` не создаёт переменную, он заполняет существующую.
2. Перед именем переменной стоит **знак `&`** (амперсанд). Это самая важная и непривычная вещь в этой строке.

## Зачем нужен знак `&`

`&` означает «**адрес** переменной». Не само значение, а место в памяти, где это значение лежит.

Аналогия: представь, что переменная `x` — это твоя квартира. У неё есть **содержимое** (мебель, ты сам) и **адрес** (улица, дом, квартира).

- Когда ты пишешь `fmt.Println(x)` — ты передаёшь функции **содержимое**: «вот моя мебель, посмотри».
- Когда ты пишешь `fmt.Scan(&x)` — ты передаёшь функции **адрес**: «вот ключи от моей квартиры, можешь зайти и поменять мебель».

`Scan` должен **изменить** твою переменную (записать туда то, что прочитал). Чтобы изменить — ему нужен доступ к самой переменной, а не к её копии. Поэтому передаём адрес.

Это первое прикосновение к концепции **указателей** в Go. Полностью разбираться с ними сейчас не нужно — достаточно правила: «когда функция должна изменить переменную, передай ей адрес через `&`».

## Несколько значений за один вызов

`Scan` умеет читать несколько значений сразу — они должны быть разделены пробелами или переводами строк:

```go
var a, b int
fmt.Scan(&a, &b)
```

Если пользователь введёт `10 20` и нажмёт Enter — `a` станет `10`, `b` станет `20`.

Если пользователь введёт `10` Enter `20` Enter — результат тот же. Scan ждёт, пока не получит все значения.

## Пример с разбором

```go
package main

import "fmt"

func main() {
    var a, b int

    fmt.Println("Введи два числа:")
    fmt.Scan(&a, &b)

    sum := a + b
    fmt.Println("Сумма:", sum)
}
```

**Что происходит при запуске:**

1. Объявляются две переменные `a` и `b` типа `int`. Сейчас в них лежат нули (Go всегда инициализирует переменные нулевым значением).
2. На экран выводится приглашение «Введи два числа:».
3. Программа **останавливается** и ждёт ввода. Это важно: Scan блокирующий — он не пойдёт дальше, пока пользователь что-то не введёт.
4. Пользователь печатает, например, `10 20` и жмёт Enter.
5. Scan читает эти два числа и записывает их в `a` и `b` по их адресам.
6. Программа считает сумму и выводит результат.

## Чтение строк

Для чисел используется `Scan`, но что если нам нужна **строка** — например, операция `+` или `-`?

```go
var op string
fmt.Scan(&op)
```

Тоже работает, но с подвохом: `Scan` читает до первого пробела. Если пользователь введёт `hello world`, в `op` попадёт только `hello`. Для нашего калькулятора это нормально — операция всегда односимвольная.
