# Модуль 2. Ввод от пользователя — подсказки

> ⚠️ Открывай только после 30 минут самостоятельной борьбы.

## Уровень 1: наводящие вопросы

1. Какая функция в пакете `fmt` читает данные от пользователя?
2. Что нужно передать этой функции, чтобы она положила прочитанное в твою переменную?
3. Можно ли использовать `:=` для переменной, в которую будет писать `Scan`? Подумай: что должно существовать **раньше** — переменная или вызов `Scan`?

## Уровень 2: про объявление переменных под Scan

`Scan` заполняет **существующую** переменную. Значит, переменная должна быть создана до вызова. Но `:=` создаёт переменную **сразу со значением**. Какое значение положить в неё, если значения ещё нет?

Тут как раз нужна форма `var`:

```go
var a float64
```

Это создаёт переменную `a` со значением по умолчанию (для чисел — `0`). Дальше её можно передать в `Scan`.

## Уровень 3: про знак `&`

Если ты забыл `&`, программа не скомпилируется. Сообщение будет примерно такое:

```
cannot use a (type float64) as type *float64 in argument to fmt.Scan
```

Расшифровка: «ты передал значение типа `float64`, а нужен **указатель** на `float64` (это `*float64`)». Знак `&` как раз превращает значение в указатель.

Не пытайся пока разбираться с типом `*float64` глубоко — достаточно правила: «`Scan` всегда требует `&` перед именем переменной».

## Уровень 4: про выбор операции

Если у тебя получился такой код и ты не понимаешь, почему он не работает:

```go
if op == '+' {
    ...
}
```

Подсказка: в Go одинарные кавычки `'+'` — это **символ** (тип `rune`), а двойные `"+"` — это **строка** (тип `string`). Они разные типы. `fmt.Scan(&op)` для `var op string` читает строку, поэтому сравнивать нужно со строкой: `op == "+"`.

## Уровень 5: про эксперименты

**Эксперимент с буквой вместо числа.** Когда ты вводишь `abc` вместо числа, `Scan` возвращает ошибку, но ты её **не проверяешь** — поэтому в переменной остаётся 0, и программа «как ни в чём не бывало» считает с нулём. Это не баг, это последствие игнорирования возвращаемого значения. В модуле 4 мы научимся это ловить.

**Эксперимент с делением на ноль.** Для `float64` Go не падает! Он возвращает специальное значение `+Inf` (бесконечность) или `NaN` (не число). Попробуй вывести результат — увидишь странное слово. Для `int` поведение другое — там программа упадёт с паникой. Это и есть одна из причин, почему мы взяли `float64`.
