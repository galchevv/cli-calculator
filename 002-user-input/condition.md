# Модуль 2. Ввод от пользователя — условие

## Задание

Перепиши свой калькулятор так, чтобы он **спрашивал** у пользователя два числа и операцию, а потом выводил результат.

## Что должна делать программа

Пример сессии:

```
Введи первое число: 20
Введи второе число: 4
Введи операцию (+, -, *, /): +
Результат: 24
```

## Требования

1. Программа спрашивает два числа и одну операцию через `fmt.Scan`.
2. Перед каждым вводом — приглашение через `fmt.Println` или `fmt.Print`.
3. Поддерживаются 4 операции: `+`, `-`, `*`, `/`.
4. Используй `if` или `switch` для выбора операции.
5. Числа храни как `float64` — чтобы деление работало корректно.
6. Результат выведи с понятной подписью.

## Подсказка по выбору операции

`if` для четырёх вариантов будет некрасивым:

```go
if op == "+" {
    result = a + b
} else if op == "-" {
    result = a - b
} else if ...
```

Сейчас можно так — нормальный первый шаг. В следующих модулях перепишем на `switch`.

## Шаги

1. Возьми код из модуля 1 как основу
2. Замени захардкоженные числа на объявления переменных без значений (`var a float64`)
3. Добавь `fmt.Scan(&a)` для каждого числа
4. Добавь чтение строки операции (`var op string`, `fmt.Scan(&op)`)
5. Реализуй выбор операции через `if` или `switch`
6. Выведи результат

## Эксперименты после того, как всё работает

1. **Запусти и введи `10 5 +`** одной строкой через пробелы — что произойдёт? Почему?
2. **Введи букву вместо числа** (например, `abc`). Что произойдёт? Что выведет программа?
3. **Введи `0` как второе число и операцию `/`**. Что произойдёт?
4. **Убери знак `&` в одном из `Scan`-вызовов.** Программа скомпилируется? Если нет — что говорит компилятор?

Эти эксперименты создадут «болевые точки», которые мы будем лечить в модуле 4 (обработка ошибок). Сейчас — просто наблюдай и запоминай.
