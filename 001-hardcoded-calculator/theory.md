# Модуль 1. Захардкоженный калькулятор — теория

## Контекст

Прежде чем калькулятор начнёт принимать данные от пользователя, он должен научиться **хранить** числа и **выполнять** с ними операции. Это два самых базовых навыка любой программы: положить значение в память и сделать с ним что-то полезное.

В этом модуле числа будут жёстко прописаны прямо в коде — это нормально для первого шага. Сначала научимся считать, потом — принимать ввод.

## Ядро: переменные и типы

**Переменная** — это именованная ячейка памяти, в которой лежит значение. У каждой переменной есть **тип** — он говорит компилятору, какого вида значения там можно хранить и что с ними можно делать.

В Go типы строгие: если ты сказал «эта переменная — целое число», ты не сможешь положить туда строку. Это хорошо — компилятор будет ловить твои ошибки заранее.

Базовые числовые типы, которые тебе понадобятся прямо сейчас:

| Тип | Что хранит | Пример значений |
|-----|------------|------------------|
| `int` | целые числа | `-3`, `0`, `42`, `1000` |
| `float64` | дробные числа | `3.14`, `-0.5`, `2.0` |

## Почему так

В большинстве «динамических» языков (Python, JavaScript) тип переменной определяется во время выполнения. В Go — на этапе компиляции. Это даёт два важных свойства:

1. **Скорость.** Компилятор знает заранее, сколько места занимает переменная и какие операции с ней допустимы — программа работает быстрее.
2. **Надёжность.** Многие ошибки ловятся **до запуска**. Если ты случайно попытался сложить число со строкой — программа даже не скомпилируется.

Это и причина, по которой Go так популярен на бэкенде: для серверов важна и скорость, и предсказуемость.

## Структура: как объявить переменную

В Go есть два способа объявления:

```go
// Способ 1: явное объявление с типом
var x int = 10
var y float64 = 3.14

// Способ 2: короткое объявление (тип выводится автоматически)
x := 10        // компилятор поймёт, что это int
y := 3.14      // компилятор поймёт, что это float64
```

Короткий синтаксис `:=` используется чаще — он короче и идиоматичнее. Длинный `var` нужен, когда тип неочевиден или когда переменная объявляется без значения.

## Арифметические операции

С числами можно делать то, что ожидаешь:

```go
a := 10
b := 3

sum := a + b      // 13
diff := a - b     // 7
prod := a * b     // 30
quot := a / b     // 3  (внимание!)
rem := a % b      // 1  (остаток от деления)
```

**Важная ловушка:** деление двух `int` даёт `int`. То есть `10 / 3` = `3`, а не `3.33`. Если хочешь дробный результат — нужно работать с `float64`:

```go
a := 10.0
b := 3.0
result := a / b   // 3.3333...
```

## Вывод на экран

Чтобы увидеть результат, используй пакет `fmt`:

```go
package main

import "fmt"

func main() {
    a := 10
    b := 3
    sum := a + b
    fmt.Println("Сумма:", sum)
}
```

`fmt.Println` печатает аргументы через пробел и переводит строку в конце.

## Полный пример с разбором

```go
package main

import "fmt"

func main() {
    x := 15
    y := 4

    fmt.Println("x =", x)
    fmt.Println("y =", y)
    fmt.Println("x + y =", x+y)
    fmt.Println("x / y =", x/y)
}
```

**Что здесь происходит построчно:**

1. `package main` — каждая исполняемая программа в Go начинается с этой строки. Она говорит: «это главный пакет, его можно запустить».
2. `import "fmt"` — подключаем пакет для форматированного вывода.
3. `func main()` — точка входа в программу. С неё начинается выполнение.
4. `x := 15` — создаём переменную `x` типа `int` со значением 15.
5. `fmt.Println(...)` — печатаем строки и значения.

Запустить так: `go run main.go`
