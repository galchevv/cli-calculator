# Модуль 5. Расширение операций — условие

## Задание

Перепиши свой калькулятор так, чтобы он был **разложен на функции**, и добавь две новые операции: остаток от деления (`%`) и возведение в степень (`^`).

## Что должна делать программа

```bash
$ go run main.go 20 + 4
Результат: 24

$ go run main.go 10 % 3
Результат: 1

$ go run main.go 2 "^" 10
Результат: 1024

$ go run main.go 7 % 0
Ошибка: остаток от деления на ноль

$ go run main.go 0 "^" 0
Результат: 1
```

## Требования

1. Все арифметические операции — **отдельные функции**: `add`, `subtract`, `multiply`, `divide`, `modulo`, `power`.
2. Функция-координатор `calculate(a, b, op) (float64, error)` — выбирает нужную операцию.
3. Функция `parseArgs(args)` — отдельно отвечает за чтение и валидацию аргументов.
4. `main` должен стать коротким — только вызовы функций и обработка ошибок.
5. Все функции, которые могут провалиться, возвращают `(результат, error)`.
6. Никаких глобальных переменных.
7. Поведение для всех случаев из модуля 4 продолжает работать.

## Подсказки по реализации

**Остаток от деления для float64.** Оператор `%` в Go работает только с целыми. Для `float64` есть функция `math.Mod`:

```go
import "math"

result := math.Mod(7, 3)  // 1
```

Не забудь обработать `math.Mod(x, 0)` — она вернёт `NaN`, а не ошибку. Деление на ноль для модуля надо ловить руками.

**Возведение в степень.** Тоже из пакета `math`:

```go
result := math.Pow(2, 10)  // 1024
```

`math.Pow(0, 0)` возвращает `1` — это математическое соглашение, не баг. Можно оставить как есть.

## Шаги

1. Возьми код из модуля 4
2. Вынеси `add`, `subtract`, `multiply` в отдельные однострочные функции
3. Вынеси `divide` с проверкой на ноль в отдельную функцию, возвращающую `(float64, error)`
4. Добавь `modulo` с проверкой на ноль и `power`
5. Создай `calculate(a, b, op)` — она вызывает нужную функцию по операции
6. Создай `parseArgs(args)` — выноси туда всю работу с `os.Args` и `strconv`
7. Перепиши `main` так, чтобы там осталось только: вызов parseArgs → вызов calculate → вывод результата → обработка ошибок
8. Прогоняй каждый случай — проверь, что всё работает

## Эксперименты

1. **Посчитай длину `main`** в строках до и после рефакторинга. Запиши разницу.
2. **Попробуй прочитать `main` вслух.** Звучит как описание алгоритма? Или как набор технических деталей?
3. **Сломай нарочно одну функцию** — например, в `multiply` поменяй `a * b` на `a + b`. Запусти тесты вручную. Заметь: ты испортил **только одну функцию**, остальные продолжают работать. Это и есть **изоляция через декомпозицию**.
4. **Добавь шестую операцию** — например, минимум из двух чисел. Сколько строк изменилось? В скольких файлах? Какие функции пришлось трогать?
