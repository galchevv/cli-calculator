## Уровень 2: про синтаксис функций

Если ты не помнишь, как объявлять функцию — вот форма:

```go
func имяФункции(параметр1 тип1, параметр2 тип2) типВозврата {
    // тело
    return значение
}
```

Для нескольких возвращаемых значений — скобки:

```go
func имя(...) (тип1, тип2) {
    return значение1, значение2
}
```

Если несколько параметров одного типа — короткая запись:

```go
// длинно
func add(a float64, b float64) float64

// коротко (то же самое)
func add(a, b float64) float64
```

## Уровень 3: про выбор «возвращать ошибку или нет»

Простое правило: **функция возвращает error, если у неё есть хотя бы один способ провалиться по содержательной причине**.

- `add(2, 3)` — провалиться не может. Просто возвращает `float64`.
- `divide(2, 0)` — может провалиться (деление на ноль). Возвращает `(float64, error)`.
- `modulo(7, 0)` — может провалиться. Возвращает `(float64, error)`.
- `power(2, 10)` — провалиться не может. Просто `float64`.

Не выдумывай ошибки там, где их нет. Это и засоряет код, и сбивает с толку — читатель будет проверять `err`, который никогда не будет не-nil.

## Уровень 4: про parseArgs

Самая частая трудность — **сколько значений возвращает parseArgs**. У тебя четыре «выхода»: первое число, операция, второе число, ошибка. Пиши так:

```go
func parseArgs(args []string) (float64, string, float64, error) {
    // ...
}
```

И вызов:

```go
a, op, b, err := parseArgs(os.Args)
if err != nil { ... }
```

Если четыре значения кажутся перебором — это нормальный сигнал, что в более серьёзном проекте ты бы создал **структуру** и вернул её. Но для калькулятора структура — это уже оверинжиниринг. В следующих проектах увидишь, когда она оправдана.

## Уровень 5: про передачу слайса

Заметь, что `parseArgs` принимает `args []string`, а не читает `os.Args` напрямую. Это **важно**: функция не должна знать, **откуда** взялись данные. Это называется **чистая функция** — её результат зависит только от входов.

Преимущества:
- Можно протестировать, передав любой слайс
- Можно переиспользовать (например, читать из файла, а не из аргументов)
- Легче читать — все зависимости функции видны в её сигнатуре

## Уровень 6: про math.Mod и math.Pow

Если ты пытаешься писать `7 % 3` для `float64` — компилятор скажет:

```
invalid operation: 7 % 3 (operator % not defined on untyped float)
```

Оператор `%` в Go работает только с целыми. Для `float64` — функция `math.Mod`:

```go
import "math"

result := math.Mod(7, 3)  // 1
```

Не забудь добавить `"math"` в импорты.

`math.Pow(основание, показатель)` — степень. Тоже из `math`.

## Уровень 7: про проверку нуля для math.Mod

`math.Mod(x, 0)` не паникует и не возвращает ошибку — она возвращает `NaN`. Если ты не проверишь явно, программа выведет `Результат: NaN`, и пользователь не поймёт, что произошло. Проверь до вызова:

```go
func modulo(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("остаток от деления на ноль")
    }
    return math.Mod(a, b), nil
}
```
